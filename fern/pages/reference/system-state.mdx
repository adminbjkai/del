## 1. Executive summary

- **236** top-level directories under `/apps` (249G total). Of these, roughly **114
  are LIVE** apps, **29 DISABLED** (valid config, cleanly stopped), **4
  MISCONFIGURED**, and the rest (**~89**) are NOT-CONFIGURED / NOT-AN-APP / a
  handful ABANDONED — see the exact per-state counts below.
- The 70 "no DEL association" directories are mostly **intentional data/reference
  storage, not gaps** — the large ones (`models` 49G, `LTX-2` 7.7G, `ideogram` 7.1G,
  `arch` 3.1G) were never meant to be services.
- **Docker:** 180 containers (179 running), 166 images (3 dangling-but-in-use, 3
  truly unused), 98 volumes (21 orphaned, ~10.3GB), 88 networks (1 genuinely
  cross-app-shared: `rowboat_net`).
- **Nginx:** 135 enabled sites (100% symlinks, 0 debris), 137 available files — DEL's
  inventory matches disk exactly.
- **Ports:** 179 listening sockets; 155 internal, 22 distinct public-bound ports
  (mostly intentional; one known exception — samba).
- **134 live subdomains**, now regenerated from the current scan.
- **Systemd:** 75 custom unit files, **0 failed units** (2 were failed pre-audit,
  both resolved — see [What was fixed](#6-what-was-fixed-in-this-audit)).

<Callout intent="success">
  **Verdict:** the system is clean and healthy. Docker, nginx, and DEL's own
  inventory all reconcile to ground truth with zero real discrepancies beyond
  expected scan-lag drift. The known exceptions are narrow and fully enumerated in
  [§5](#5-known-exceptions--needs-attention): 8 nginx sites serving 502 for
  deliberately-stopped apps, 3 genuinely misconfigured apps, a couple of incomplete
  decommissions, a few apps DEL's scanner doesn't yet track, and one pre-existing
  security note (samba public bind). None are active incidents.
</Callout>

## 4. Docker / nginx / ports health

**Docker:** 180 containers (179 running, 1 exited but owned by the still-active
`blinko` app — needs a restart investigation, not removal). 166 images, 3 dangling
all correctly in-use shared DB layers, 3 truly-unused safe to prune (~1.9GB). 98
volumes, 21 orphaned (~10.3GB) — largest was `openshell-cluster-nemoclaw` (4.7GB,
**already deleted**, see [§6](#6-what-was-fixed-in-this-audit)) and
`netdata_netdatacache` (3.4GB, netdata stopped but volume not removed). 88 networks,
only `rowboat_net` a genuine cross-app share. Build cache: 16.44GB, 100% reclaimable
— the single largest system-wide reclaim opportunity.

**Nginx:** 135 sites-enabled (100% symlinks, 0 broken links), 137 sites-available, 0
debris files. DEL's inventory (272 rows) matches disk exactly. 0 duplicate
server_names, 0 real port conflicts (all multi-domain-per-port cases are documented
aliases). 8 dead-upstream enabled sites — all confirmed intentionally-stopped apps,
not orphaned configs.

**Ports:** 179 listening sockets, 155 internal, 22 distinct public-bound ports.
Public-bound is mostly intentional (ssh, http/s, streaming, STUN/TURN, NoMachine, PCP
metrics, RustDesk, AionUI, a few public-by-design frontends) plus **samba (139/445)**
— the one pre-existing security exception. Cockpit's prior public exposure is
**confirmed resolved** (now loopback-only).

**DEL-inventory-accuracy verdict:** matches ground truth on every count checked
across containers, images, volumes, networks, ports, and systemd attribution — the
only drift anywhere in the entire audit is a 1-image scan-timing lag.

## 7. How to refresh this

1. **Re-scan via DEL** — trigger a rescan from the web UI or `del-admin rescan`.
2. **Regenerate the port registry** — `/apps/del/scripts/gen-registry.py` against the
   latest scan id; run this after every rescan.
3. **Raw audit data** lives at `/apps/del/data/audit/2026-07-20/` (`filesystem.md`,
   `nginx.md`, `docker.md`, `ports-services.md`, `dirs-A.md`, `dirs-B.md`,
   `dirs-C.md`, `shared-and-config.md`). Re-running the same 8-lane methodology
   periodically is the recommended way to keep this reference current.

## Port provenance & true exposure

All 172 listening ports were traced end-to-end (listener → pid → exact binary/script → launcher → config file → working dir), **0 unexplained**: ~110 Docker-compose (traced to the exact compose file, including nested paths), ~35 systemd units, 3 host-network containers, the rest system daemons and a few manual/orphan processes.

<Callout intent="info">
**ufw default-DENY is the real security gate.** The host runs `ufw` active with policy DROP, so a process binding `0.0.0.0` is **not** publicly reachable unless ufw explicitly allows its port. Ports flagged by bind-address in past audits — Samba 139/445, the dev servers (3015/3019/3020/3901), AionUi 9000, PCP 44321, and DEL's own docs server (8072/8073) — are all **blocked externally**. Only the deliberate ufw ALLOW rules are genuinely reachable.
</Callout>

Manual/orphan processes (functioning, unsupervised): 17imgshare gunicorn (8087), `/apps/htmls` node (8195), claude-code-router (3456), adb (5037), and the anisette/netmuxd containers (bare `docker run`).
